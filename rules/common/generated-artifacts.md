# Generated Artifact Rules

## Covered Rules

- `GEN-1.1`: Generated source artifacts are not accepted as hand-maintained source. This catches common generated markers such as `@generated`, `<auto-generated>`, and `Generated by`.
- `GEN-1.2`: Generated output folders must not be committed as source. This catches paths such as `output/`, `test-results/`, and `playwright-report/` when they enter the scan scope.
- `GEN-2.1`: Generated directories must be ignored unless the profile explicitly tracks them.
- `GEN-2.2`: Generated files need source owner and generator provenance.
- `GEN-2.3`: Generated files cannot be edited manually; regenerate them from the source owner.
- `GEN-2.4`: Generated contract artifacts must carry a source schema or contract hash.
- `GEN-2.5`: Generated schema files must be reproducible byte-for-byte with deterministic hash metadata.
- `GEN-2.6`: `coverage/`, `playwright-report/`, `test-results/`, and `output/` cannot be tracked as source.
- `GEN-2.7`: Generated files cannot claim to be the single source of truth.
- `GEN-2.8`: Generated code cannot contain validation suppressions.
- `GEN-2.9`: Generated code cannot live inside domain modules by default.
- `GEN-2.10`: Generated snapshots must be stable and redact timestamps, UUIDs, random IDs, and machine-local paths.

## Enforcement

Run:

```bash
ocentra-enforcer scan --root <repo> --languages common --files <changed-files>
```

Generated output belongs in ignored build folders, CI artifacts, or explicit generated-code lanes with a project-owned exception.

## Fails

- Generated files are edited by hand or committed without generator provenance.
- Generated outputs drift from source schemas, contracts, or templates.

## Passes

- Generated files have deterministic provenance and are refreshed by the configured generator.
- Architecture/source-shape checks route generated outputs through generated-artifact policy.

## Fix Recipe

1. Re-run the generator from the owning source.
2. Commit only configured generated outputs.
3. Re-run generated-artifact and contract drift checks.

## Validator

- scanner: `common/generated-artifacts`
- command: `ocentra-enforcer check generated-artifacts --root <repo> --files <generated-files>`
